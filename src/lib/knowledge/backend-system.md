# 李伟个人知识库（后端与系统能力补充）

## 后端能力概览

李伟在当前项目中具备完整的后端联调与服务端能力，主要覆盖以下方向：

- 基于 Next.js API Route 的服务端接口设计与实现
- 基于 Drizzle ORM + PostgreSQL（Supabase）进行数据建模与查询
- 认证体系（Better Auth）与会话管理（含 Google 社交登录）
- 邮件通知系统（QQ SMTP / 管理员通知 / 用户状态通知）
- AI 接口聚合（OpenAI 兼容网关、模型回退、错误降级）
- 实时与准实时能力（Supabase Realtime + 轮询兜底）
- 后台可视化统计 API 聚合（用户、评论、留言、通知）

## 当前项目中的后端模块

### 1. 认证与登录

- 使用 Better Auth 管理登录、注册、会话与社交登录。
- 支持 Google 登录（`/api/auth/sign-in/social` 等链路）。
- 支持角色识别与后台权限判断（admin 权限校验）。

### 2. 内容系统 API（博客/项目/作者）

- 博客：列表、详情、评论、点赞、阅读量统计。
- 项目：列表、详情。
- 作者：基础信息、技能、经历、教育、荣誉、图片。
- 支持数据库异常时的降级返回，保障前台可用性。

### 3. 留言板与通知

- 留言板支持新增、审核状态更新、删除。
- 触发链路包含：
  - 新留言提醒（管理员邮箱）
  - 留言状态更新提醒（给留言用户邮箱）
- 后台通知中心支持读取、已读、批量处理与删除。

### 4. 邮件系统

- 使用 Nodemailer 对接 QQ SMTP。
- 支持管理员通知收件人配置（数据库配置 + 环境变量回退）。
- 已覆盖事件类型：
  - `user_signup`（新用户注册）
  - `guestbook_message`（留言板新留言）
  - `blog_comment`（博客新评论）
  - `comment_reply`（评论区新回复）
  - `guestbook_reply`（留言状态更新通知）

### 5. AI 智能客服

- 聊天接口位于 `/api/chat`，对接 OpenAI 兼容 API。
- 支持模型回退（fallback model list）。
- 支持数据库存储会话（可配置禁用并进入无状态模式）。
- 知识来源为本地知识库目录（`src/lib/knowledge`）文件聚合。

### 6. 可视化大屏与后台运营

- 提供 `/api/admin/visual` 聚合数据：
  - 用户总量
  - 博客总量
  - 项目总量
  - 评论总量
  - 留言总量
  - 未读通知数
  - 近 7 日趋势与事件分布
- 后台通知中心联调同一数据源，支持实时刷新与轮询。

## 工程实践能力

- 接口异常兜底：在数据库不可用时保证前台核心路由可浏览。
- 环境变量管理：区分开发、预发布、生产配置并支持快速切换。
- 部署与回归：Vercel 自动构建部署，支持线上快速验证与热修复。
- 可观测性：关键失败链路输出服务端日志，便于定位问题。
